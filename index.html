Módulo 1 — SELECT, WHERE e Filtros

Banco: db_ecommerce
🟢 Básico
01
Listando todos os clientes ativos SELECT · WHERE

Selecione o nome, email e cidade de todos os clientes que estão ativos (ativo = 1). Ordene pelo nome em ordem alfabética.

💡 Dica: Use ORDER BY nome ASC no final da query.
▶ Ver resposta 
SELECT nome, email, cidade
FROM   clientes
WHERE  ativo = 1
ORDER BY nome ASC;
02
Produtos por faixa de preço WHERE · BETWEEN

Liste o nome_produto, categoria e preco dos produtos com preço entre R$200 e R$1.000. Ordene do mais barato para o mais caro.

💡 Dica: BETWEEN 200 AND 1000 inclui os dois extremos.
▶ Ver resposta 
SELECT nome_produto, categoria, preco
FROM   produtos
WHERE  preco BETWEEN 200 AND 1000
ORDER BY preco ASC;
03
Pedidos com status específico WHERE · IN

Encontre todos os pedidos com status "Cancelado" ou "Processando". Mostre o pedido_id, cliente_id, data_pedido, status e total.

💡 Dica: O operador IN é mais elegante do que múltiplos OR.
▶ Ver resposta 
SELECT pedido_id, cliente_id, data_pedido, status, total
FROM   pedidos
WHERE  status IN ('Cancelado', 'Processando')
ORDER BY data_pedido;
04
Busca por texto parcial LIKE

Liste todos os produtos da categoria "Informática" cujo nome começa com a letra "M" ou contém a palavra "HD".

💡 Dica: Use LIKE 'M%' para "começa com M" e LIKE '%HD%' para contém.
▶ Ver resposta 
SELECT nome_produto, categoria, preco
FROM   produtos
WHERE  categoria = 'Informática'
  AND  (nome_produto LIKE 'M%' OR nome_produto LIKE '%HD%');
05
Pedidos de 2023 DATE · YEAR()

Liste todos os pedidos realizados no ano de 2023. Mostre o pedido_id, data_pedido, status e total, ordenando pela data mais recente.

💡 Dica: Use a função YEAR(data_pedido) = 2023 ou BETWEEN '2023-01-01' AND '2023-12-31'.
▶ Ver resposta 
SELECT pedido_id, data_pedido, status, total
FROM   pedidos
WHERE  YEAR(data_pedido) = 2023
ORDER BY data_pedido DESC;

Módulo 2 — Funções Agregadas e GROUP BY

Banco: db_ecommerce · db_rh
🟢 Básico–Médio
06
Total de pedidos por status COUNT · GROUP BY

Conte quantos pedidos existem para cada status. Mostre o status e a quantidade, ordenando do maior para o menor número de pedidos.

▶ Ver resposta 
SELECT   status, COUNT(*) AS qtd_pedidos
FROM     pedidos
GROUP BY status
ORDER BY qtd_pedidos DESC;
07
Receita total por método de pagamento SUM · GROUP BY

Calcule a receita total (soma do campo total) agrupada por metodo_pagamento. Considere apenas pedidos com status "Entregue".

💡 Dica: O WHERE filtra antes do GROUP BY. Não confunda com HAVING!
▶ Ver resposta 
SELECT   metodo_pagamento,
         SUM(total) AS receita_total
FROM     pedidos
WHERE    status = 'Entregue'
GROUP BY metodo_pagamento
ORDER BY receita_total DESC;
08
Preço médio por categoria (db_ecommerce) AVG · ROUND · HAVING

Calcule o preço médio dos produtos por categoria. Mostre apenas categorias com preço médio acima de R$500. Arredonde para 2 casas decimais.

💡 Dica: Use HAVING AVG(preco) > 500 — o HAVING filtra após a agregação.
▶ Ver resposta 
SELECT   categoria,
         ROUND(AVG(preco), 2) AS preco_medio
FROM     produtos
GROUP BY categoria
HAVING   AVG(preco) > 500
ORDER BY preco_medio DESC;
09
Salário médio por departamento (db_rh) AVG · GROUP BY · JOIN

No banco db_rh, mostre o nome do departamento, o número de funcionários ativos e o salário médio. Ordene pelo salário médio decrescente.

▶ Ver resposta 
SELECT   d.nome_depto,
         COUNT(f.func_id)     AS qtd_funcionarios,
         ROUND(AVG(f.salario), 2) AS salario_medio
FROM     departamentos  d
JOIN     funcionarios   f ON d.depto_id = f.depto_id
WHERE    f.ativo = 1
GROUP BY d.depto_id, d.nome_depto
ORDER BY salario_medio DESC;
10
Produtos mais vendidos (quantidade) SUM · GROUP BY · LIMIT

Usando a tabela itens_pedido, descubra os 5 produtos mais vendidos em quantidade total. Mostre o produto_id e o total de unidades vendidas.

💡 Dica: LIMIT 5 vai no final da query, após o ORDER BY.
▶ Ver resposta 
SELECT   produto_id,
         SUM(quantidade) AS total_vendido
FROM     itens_pedido
GROUP BY produto_id
ORDER BY total_vendido DESC
LIMIT    5;

Módulo 3 — JOINs

Banco: db_ecommerce · db_rh
🟡 Médio
11
Pedidos com nome do cliente INNER JOIN

Combine as tabelas pedidos e clientes para mostrar: pedido_id, nome do cliente, data_pedido, status e total. Filtre apenas pedidos "Entregue".

▶ Ver resposta 
SELECT  p.pedido_id,
        c.nome AS cliente,
        p.data_pedido,
        p.status,
        p.total
FROM    pedidos   p
JOIN    clientes  c ON p.cliente_id = c.cliente_id
WHERE   p.status = 'Entregue'
ORDER BY p.data_pedido DESC;
12
Detalhes completos do pedido (3 tabelas) JOIN · JOIN · JOIN

Faça um relatório juntando pedidos, itens_pedido e produtos. Mostre: pedido_id, data_pedido, nome_produto, quantidade e preco_unitario.

▶ Ver resposta 
SELECT  p.pedido_id,
        p.data_pedido,
        pr.nome_produto,
        ip.quantidade,
        ip.preco_unitario
FROM    pedidos       p
JOIN    itens_pedido  ip ON p.pedido_id  = ip.pedido_id
JOIN    produtos      pr ON ip.produto_id = pr.produto_id
ORDER BY p.pedido_id, pr.nome_produto;
13
Clientes sem pedidos (LEFT JOIN) LEFT JOIN · IS NULL

Identifique quais clientes nunca fizeram nenhum pedido. Mostre o nome e email desses clientes.

💡 Dica: Use LEFT JOIN e filtre onde o pedido_id IS NULL — isso indica que o cliente não tem correspondência na tabela de pedidos.
▶ Ver resposta 
SELECT  c.nome, c.email
FROM    clientes c
LEFT JOIN pedidos p ON c.cliente_id = p.cliente_id
WHERE   p.pedido_id IS NULL;
14
Funcionários e seus gestores (Self JOIN) SELF JOIN · db_rh

No banco db_rh, liste cada funcionário com o nome do seu gestor. Funcionários sem gestor (diretores) devem aparecer com "Sem gestor".

💡 Dica: Faça um LEFT JOIN da tabela funcionarios com ela mesma, usando gestor_id para cruzar com func_id.
▶ Ver resposta 
SELECT  f.nome                              AS funcionario,
        f.cargo,
        COALESCE(g.nome, 'Sem gestor')   AS gestor
FROM    funcionarios f
LEFT JOIN funcionarios g ON f.gestor_id = g.func_id
WHERE   f.ativo = 1
ORDER BY gestor, f.nome;
15
Receita total por cliente JOIN · SUM · GROUP BY

Mostre o nome e estado de cada cliente junto com o total gasto em pedidos "Entregue". Ordene pelo total gasto decrescente. Inclua apenas clientes com gasto acima de R$1.000.

▶ Ver resposta 
SELECT   c.nome, c.estado,
         SUM(p.total) AS total_gasto
FROM     clientes c
JOIN     pedidos  p ON c.cliente_id = p.cliente_id
WHERE    p.status = 'Entregue'
GROUP BY c.cliente_id, c.nome, c.estado
HAVING   SUM(p.total) > 1000
ORDER BY total_gasto DESC;

Módulo 4 — Subqueries e CTEs

Banco: db_ecommerce · db_rh
🟡 Médio
16
Produtos com preço acima da média Subquery no WHERE

Liste os produtos cujo preço está acima da média geral de preços. Mostre o nome, categoria e preço.

💡 Dica: A subquery (SELECT AVG(preco) FROM produtos) retorna um único valor que pode ser usado no WHERE.
▶ Ver resposta 
SELECT nome_produto, categoria, preco
FROM   produtos
WHERE  preco > (SELECT AVG(preco) FROM produtos)
ORDER BY preco DESC;
17
Clientes que compraram Notebooks Subquery com IN

Encontre os clientes que já compraram o produto "Notebook Dell". Mostre o nome e email.

💡 Dica: Você precisará de subqueries aninhadas: clientes → pedidos → itens_pedido → produto.
▶ Ver resposta 
SELECT nome, email
FROM   clientes
WHERE  cliente_id IN (
    SELECT cliente_id
    FROM   pedidos
    WHERE  pedido_id IN (
        SELECT pedido_id
        FROM   itens_pedido
        WHERE  produto_id = (
            SELECT produto_id
            FROM   produtos
            WHERE  nome_produto = 'Notebook Dell'
        )
    )
);
18
Funcionários acima da média salarial do depto (db_rh) Subquery correlacionada

Liste os funcionários cujo salário está acima da média do seu próprio departamento. Mostre nome, cargo, departamento e salário.

💡 Dica: Uma subquery correlacionada referencia a tabela externa com um alias: WHERE f2.depto_id = f1.depto_id.
▶ Ver resposta 
SELECT  f1.nome, f1.cargo, d.nome_depto, f1.salario
FROM    funcionarios f1
JOIN    departamentos d ON f1.depto_id = d.depto_id
WHERE   f1.salario > (
    SELECT AVG(f2.salario)
    FROM   funcionarios f2
    WHERE  f2.depto_id = f1.depto_id
)
AND f1.ativo = 1
ORDER BY d.nome_depto, f1.salario DESC;
19
Top produtos por receita usando CTE WITH (CTE)

Use uma CTE (Common Table Expression) para calcular a receita de cada produto (quantidade × preco_unitario em itens_pedido) e depois mostre apenas os 3 com maior receita, com o nome do produto.

💡 Dica: CTEs tornam o código mais legível: WITH nome_cte AS (SELECT ...) e depois use como se fosse uma tabela.
▶ Ver resposta 
WITH receita_produto AS (
    SELECT   produto_id,
             SUM(quantidade * preco_unitario) AS receita
    FROM     itens_pedido
    GROUP BY produto_id
)
SELECT  p.nome_produto,
        rp.receita
FROM    receita_produto  rp
JOIN    produtos         p ON rp.produto_id = p.produto_id
ORDER BY rp.receita DESC
LIMIT 3;
20
Relatório de avaliações com CTE (db_rh) WITH · AVG · JOIN

Com uma CTE, calcule a nota média por funcionário e em seguida mostre o nome, cargo e média, apenas para quem tem média de avaliação ≥ 8.

▶ Ver resposta 
WITH media_aval AS (
    SELECT   func_id,
             ROUND(AVG(nota), 1) AS media_nota
    FROM     avaliacoes
    GROUP BY func_id
)
SELECT  f.nome, f.cargo, ma.media_nota
FROM    media_aval     ma
JOIN    funcionarios  f ON ma.func_id = f.func_id
WHERE   ma.media_nota >= 8
ORDER BY ma.media_nota DESC;

Módulo 5 — Window Functions e Análises Avançadas

Banco: db_ecommerce · db_rh
🔴 Avançado
21
Ranking de funcionários por salário no depto (db_rh) RANK() OVER (PARTITION BY)

Para cada funcionário ativo, exiba seu nome, departamento, salário e uma coluna rank_salario indicando sua posição salarial dentro do departamento (1 = maior salário).

💡 Dica: RANK() OVER (PARTITION BY depto_id ORDER BY salario DESC) reinicia o ranking a cada departamento.
▶ Ver resposta 
SELECT  f.nome,
        d.nome_depto,
        f.salario,
        RANK() OVER (
            PARTITION BY f.depto_id
            ORDER BY f.salario DESC
        ) AS rank_salario
FROM    funcionarios  f
JOIN    departamentos d ON f.depto_id = d.depto_id
WHERE   f.ativo = 1
ORDER BY d.nome_depto, rank_salario;
22
Receita acumulada de pedidos por mês SUM() OVER · DATE_FORMAT

Calcule a receita mensal e a receita acumulada mês a mês de todos os pedidos "Entregue" em 2023. Mostre: mês, receita do mês e receita acumulada.

💡 Dica: Use DATE_FORMAT(data_pedido, '%Y-%m') para agrupar por mês, e SUM(...) OVER (ORDER BY mes) para o acumulado.
▶ Ver resposta 
WITH mensal AS (
    SELECT   DATE_FORMAT(data_pedido, '%Y-%m') AS mes,
             SUM(total) AS receita_mes
    FROM     pedidos
    WHERE    status = 'Entregue'
      AND    YEAR(data_pedido) = 2023
    GROUP BY DATE_FORMAT(data_pedido, '%Y-%m')
)
SELECT  mes,
        receita_mes,
        SUM(receita_mes) OVER (ORDER BY mes) AS receita_acumulada
FROM    mensal
ORDER BY mes;
23
Comparação de avaliação: ano atual vs anterior (db_rh) LAG() OVER

Para cada funcionário com avaliação em 2022 e 2023, compare a nota dos dois anos usando LAG(). Mostre: nome, nota 2022, nota 2023 e a variação (2023 − 2022).

💡 Dica: LAG(nota) OVER (PARTITION BY func_id ORDER BY ano) traz a nota do ano anterior.
▶ Ver resposta 
WITH notas AS (
    SELECT  func_id, ano, nota,
            LAG(nota) OVER (
                PARTITION BY func_id ORDER BY ano
            ) AS nota_anterior
    FROM    avaliacoes
)
SELECT  f.nome,
        n.nota_anterior  AS nota_2022,
        n.nota           AS nota_2023,
        n.nota - n.nota_anterior AS variacao
FROM    notas            n
JOIN    funcionarios   f ON n.func_id = f.func_id
WHERE   n.ano = 2023
  AND   n.nota_anterior IS NOT NULL
ORDER BY variacao DESC;
24
Percentual de cada produto sobre a receita total SUM() OVER() · Percentual

Para cada produto vendido, calcule a receita total e o percentual que representa sobre a receita geral de todos os produtos. Mostre nome, receita e % participação.

▶ Ver resposta 
WITH receitas AS (
    SELECT   produto_id,
             SUM(quantidade * preco_unitario) AS receita
    FROM     itens_pedido
    GROUP BY produto_id
)
SELECT  p.nome_produto,
        r.receita,
        ROUND(
            r.receita * 100.0 /
            SUM(r.receita) OVER ()
        , 2) AS pct_participacao
FROM    receitas  r
JOIN    produtos  p ON r.produto_id = p.produto_id
ORDER BY r.receita DESC;
25
🏆 Desafio Final — Dashboard de Clientes CTE · JOIN · Window · Condicional

Crie um relatório completo dos clientes ativos com:

  • Nome e estado do cliente
  • Quantidade total de pedidos
  • Ticket médio (média do campo total)
  • Valor do último pedido
  • Classificação: "VIP" se gasto total > R$3.000, caso contrário "Regular"

Ordene pelos clientes VIP primeiro, depois pelo maior gasto total.

💡 Dica: Use uma CTE para agregar os dados de pedidos e depois junte com clientes. Use CASE WHEN para a classificação.
▶ Ver resposta 
WITH stats_cliente AS (
    SELECT   cliente_id,
             COUNT(*) AS qtd_pedidos,
             ROUND(AVG(total), 2) AS ticket_medio,
             SUM(total)         AS gasto_total,
             MAX(total)         AS ultimo_valor
    FROM     pedidos
    WHERE    status = 'Entregue'
    GROUP BY cliente_id
)
SELECT  c.nome,
        c.estado,
        sc.qtd_pedidos,
        sc.ticket_medio,
        sc.ultimo_valor,
        sc.gasto_total,
        CASE
            WHEN sc.gasto_total > 3000 THEN 'VIP'
            ELSE 'Regular'
        END AS classificacao
FROM    clientes      c
JOIN    stats_cliente sc ON c.cliente_id = sc.cliente_id
WHERE   c.ativo = 1
ORDER BY classificacao ASC, gasto_total DESC;
-- 'VIP' vem antes de 'Regular' em ASC alfabético ✓
